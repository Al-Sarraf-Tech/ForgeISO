# ForgeISO — Assurance Overview

This document describes the CI/CD pipeline, security scanning, testing, and
compliance controls that protect the ForgeISO codebase.

---

## 7-Stage Parallel Docker CI Pipeline

ForgeISO runs a 7-stage CI pipeline on every push and pull request. All stages
execute simultaneously on separate GitHub-hosted runners using ephemeral Docker
containers. Every stage must pass before the release job can proceed.

| Stage | Name | What It Does |
|---|---|---|
| C1 | Rust (fmt, clippy, test) | `cargo fmt --check`, `cargo clippy -D warnings`, `cargo test --workspace` |
| C2 | SBOM + Audit | `cargo-deny check`, `cargo-audit`, `syft` SBOM generation |
| C3 | GUI | Builds `forge-gui` (egui) and legacy Tauri GUI (`npm ci`, `npm run build`) |
| C4 | Security | Container image scanning with `trivy`, `syft`, and `grype` |
| C5 | Integration | Full `cargo build --release`, inject smoke test with real ISO logic |
| C6 | E2E Smoke | QEMU boot test of a generated ISO (when KVM is available) |
| C7 | Lint | Fast-fail `cargo fmt --check` + `cargo clippy -D warnings` (no test compile) |

Additionally, a **Shell Lint** job runs `shellcheck` and `shfmt` against all 28
project shell scripts in `scripts/` and `.github/scripts/`.

### Pipeline Properties

- **fail-fast: false** — all stages run to completion even if one fails, giving
  a full report every time.
- **Ephemeral containers** — each stage runs in a fresh `--rm` Docker container.
  Volumes are destroyed after every run. No state leaks between stages.
- **Per-stage caching** — Cargo registry/git/target caches are keyed by stage ID
  so compiled artifacts never cross-contaminate.
- **Concurrency controls** — redundant runs on the same branch are automatically
  cancelled (`cancel-in-progress: true`). Tag-triggered release runs are never
  cancelled.

## Security Scanning

| Tool | Stage | Purpose |
|---|---|---|
| cargo-deny | C2 | License compliance + advisory database checks against `deny.toml` |
| cargo-audit | C2 | RustSec advisory database audit of all dependencies |
| trivy | C4 | Container image vulnerability scanner |
| syft | C2, C4 | Software composition analysis and SBOM generation |
| grype | C4 | Vulnerability scanner against SBOM output |
| gitleaks | C4 | Secret detection across the repository history |

---

## SBOM Generation

ForgeISO generates Software Bill of Materials in two formats:

- **CycloneDX** (`sbom.cdx.json`) — generated by `syft` in C2 and at release time
- **SPDX** (`sbom.spdx.json`) — generated by `syft` at release time

Both are uploaded as CI artifacts and attached to GitHub Releases.

### Integration and E2E

- **C5 (Integration):** Builds release binaries and runs inject smoke tests
  exercising real ISO manipulation logic.
- **C6 (E2E Smoke):** Boots a generated ISO under QEMU to validate the
  unattended installer works end-to-end.

## Pre-Push Hook Enforcement

A Git `pre-push` hook at `.git/hooks/pre-push` runs the full parallel CI
pipeline locally before any push is accepted. This ensures that broken code
cannot reach the remote repository.

The hook calls `scripts/ci/run-parallel.sh`, which builds all 7 Docker CI
images and runs every stage in parallel ephemeral containers.

---

## License Compliance (deny.toml)

`deny.toml` enforces:

- **License allow-list:** only permissive licenses (MIT, Apache-2.0, BSD-*,
  ISC, Zlib, Unicode-3.0, etc.) plus explicitly approved exceptions (MPL-2.0,
  LicenseRef-Slint-Royalty-free-2.0).
- **Advisory checks:** all RustSec advisories are checked; suppressions require
  documented rationale.
- **Duplicate crate detection:** warns on duplicate versions of the same crate.

## How to Run Locally

### Full parallel CI (all 7 stages)

```bash
make ci-parallel
# or directly:
bash scripts/ci/run-parallel.sh
```

### Selected stages only

```bash
make ci-parallel CI_STAGES=c1,c7
# or:
bash scripts/ci/run-parallel.sh c1 c7
```

### Individual checks

```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
cargo deny check
```

### Shell linting (local)

```bash
# Requires: shellcheck, shfmt
find scripts .github/scripts -name '*.sh' -type f | xargs shellcheck --severity=warning --shell=bash
find scripts .github/scripts -name '*.sh' -type f | xargs shfmt -d -i 2 -ci -bn
```

### Clean up CI artifacts

```bash
make ci-clean
```
